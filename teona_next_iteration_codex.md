## 3. Названия объектов недвижимости

### Проблема

Сейчас в карточках появляются плохие названия:

```text
Все новостройки
2-комн. квартиры
Кабинет
Избранное
72 Отзывов
Ремонт в подарок Скидка до 15%...
```

Это выглядит как случайный текст со страницы сайта, а не название объекта.

### Нужно сделать

Название объекта должно формироваться из нормализованных данных.

Если объект — квартира:

```text
1-к квартира, 40.6 м² в ЖК «Nova Vita»
Студия, 27.4 м² в ЖК «Рекорд»
2-к квартира, 58 м² в ЖК «Первый»
```

Если известно расположение, можно добавлять улицу:

```text
1-к квартира на ул. Петра Метальникова
2-к квартира в районе Прикубанский
```

Если объект — дом:

```text
Дом 128 м² в КП «Мелодия»
Дом 100 м², участок 5 сот. в Немецкой деревне
Коттедж 145 м² в КП «Краснодар»
```

### Правила формирования названия

Для квартиры:

```text
{roomsLabel}, {area} м² в {complexName}
```

Приоритеты:

1. Если есть ЖК — использовать ЖК.
2. Если нет ЖК, но есть улица — использовать улицу.
3. Если нет улицы, но есть район — использовать район.
4. Если данных мало — использовать аккуратный fallback: `Квартира {area} м²`.

Для дома:

```text
Дом {houseArea} м² в {location}
```

Если есть участок:

```text
Дом {houseArea} м², участок {landArea} сот. в {location}
```

### Запрещенные названия

Нельзя использовать как title:

```text
Все новостройки
Избранное
Кабинет
Отзывы
72 Отзывов
101 Отзывов
Старт продаж
Скоро в продаже
Недорогие
Комфорт
Бизнес
Главная
Каталог
```

Если источник отдает такие слова, их надо фильтровать.

## 4. Фотографии объектов

### Проблема

Сейчас иногда тянутся логотипы, иконки, favicon или служебные картинки сайтов. Это выглядит плохо.

### Решение

В карточках списка объектов лучше вообще убрать фотографии.

Текущий список объектов должен стать более аккуратным и текстовым, без большого фото-блока.

Карточка объекта в списке:

```text
┌──────────────────────────────────────┐
│ 1-к квартира, 40.6 м² в ЖК «Nova Vita»│
│ Домострой Краснодар                  │
│                                      │
│ 5 275 000 ₽                          │
│ 1-комн. · 40.6 м² · Прикубанский     │
│                                      │
│ [Подробнее] [В подборку]             │
└──────────────────────────────────────┘
```

Для дома:

```text
┌──────────────────────────────────────┐
│ Дом 128 м² в КП «Мелодия»            │
│ КП Краснодар                         │
│                                      │
│ 9 350 000 ₽                          │
│ Дом 128 м² · участок 5 сот.          │
│                                      │
│ [Подробнее] [В подборку]             │
└──────────────────────────────────────┘
```

### Что делать с фото

1. В карточках списка фото не показывать.
2. В модальном окне объекта фото можно показывать только если оно точно является фото объекта.
3. Если фото не найдено — показывать аккуратную заглушку:

```text
Фото не найдено
Откройте источник, чтобы посмотреть изображения на сайте застройщика.
```

### Запрещено использовать как фото

Не использовать:

```text
favicon
logo
иконки меню
иконки соцсетей
svg-иконки
маленькие изображения меньше 300x200
картинки с названием logo, icon, favicon, sprite, placeholder
```

Фильтрация по URL: если image url содержит `logo`, `favicon`, `icon`, `sprite`, `placeholder`, `avatar`, `social`, `telegram`, `vk`, `whatsapp`, такую картинку не сохранять как `image_url`.

### Кнопка источника

В карточке объекта и в модальном окне должна быть возможность открыть источник:

```text
[Открыть источник]
```

Если фото в приложении нет, риелтор сможет посмотреть фото на сайте источника.

```text
В рамках дипломного проекта разрабатывается веб-приложение «Тэона» — рабочее пространство риелтора для подбора недвижимости клиентам. ```

- создание и хранение клиентов;
- ввод параметров поиска недвижимости;

- поиск объектов по заданным параметрам;
- отображение найденных объектов;
- просмотр карточки объекта;
- добавление объектов в подборку клиента;
- формирование текста подборки;
- копирование телефона клиента и текста сообщения;
- хранение данных в PostgreSQL.
```

### Оформление Word-файла

```text
Шрифт: Times New Roman
Размер основного текста: 14
Межстрочный интервал: 1.5
Выравнивание: по ширине
Заголовки: жирные
```

Если через Codex сложно создать docx напрямую, можно создать markdown и отметить, что docx будет сгенерирован на следующей итерации. Но желательно создать именно `.docx`.

## 16. Definition of Done

Итерация считается готовой, если:

1. В карточках объектов нет мусорных названий.
2. Названия формируются из характеристик объекта.
3. В списке объектов нет логотипов, favicon и случайных иконок.
4. Карточки объектов выглядят аккуратно без фото.
5. В модалке объекта фото показывается только если оно реально похоже на фото объекта.
6. Если фото нет — показывается понятная заглушка.
7. Проценты совпадения и progress-bar убраны из карточек списка.
8. Кнопка “В подборку” по умолчанию оранжевая.
9. Большой график на странице клиента убран.
10. Вместо него показаны параметры клиента.
11. Графики на главной спокойные: серые столбики + один оранжевый актуальный период.
12. Поиск проверен по квартирам и домам.
13. Есть документация по источникам парсинга.
14. Есть QA-чеклист поиска.
15. Создан Word-файл пояснительной записки.
16. Проект собирается и запускается.
17. Изменения закоммичены и запушены.
