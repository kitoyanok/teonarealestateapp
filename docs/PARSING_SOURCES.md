# Источники парсинга

## Как устроен поиск

Поиск выполняет Python `search-service`.

Источники поиска не ищутся автоматически. Они заранее описаны в коде как список стартовых URL в:

- [apps/search-service/app/adapters/sources.py](/Users/like-shockpritotskaya-event/Documents/Зубач/Codex/realestate/apps/search-service/app/adapters/sources.py)

Сейчас нет отдельного индивидуального парсера под каждый сайт. Все источники проходят через общий `BaseAdapter`: он читает JSON-LD, видимые карточки HTML и приводит данные к единому формату `PropertyItem`.

Основная базовая логика парсинга находится в:

- [apps/search-service/app/adapters/base.py](/Users/like-shockpritotskaya-event/Documents/Зубач/Codex/realestate/apps/search-service/app/adapters/base.py)

## Что происходит при поиске

1. Node API формирует payload из параметров клиента.
2. Python `search-service` получает `POST /search`.
3. В зависимости от `propertyType` выбираются адаптеры квартир или домов.
4. Каждый адаптер идет на заранее заданные стартовые URL.
5. Из HTML и JSON-LD достаются карточки объектов.
6. Карточки нормализуются:
   - формируется title
   - очищается или генерируется description
   - фильтруются плохие картинки
   - цена и площадь приводятся к числам
   - определяется город
   - определяется район города
   - извлекаются характеристики объекта
7. Объекты не из города Краснодар отбрасываются до matcher.
8. Затем matcher проверяет соответствие параметрам клиента.
9. Только валидные объекты возвращаются в Node API.

## Ограничение по географии

Поиск должен работать только по городу Краснодар.

Парсер принимает объект, если есть один из признаков:

- в тексте явно указан `г. Краснодар` или `город Краснодар`
- указан район/микрорайон города Краснодар
- стартовая ссылка источника явно относится к Краснодару

Парсер отбрасывает объект, если в тексте найден другой населенный пункт Краснодарского края, например:

- Сочи
- Анапа
- Новороссийск
- Геленджик
- Армавир
- Ейск
- Туапсе
- Горячий Ключ
- Яблоновский
- Динская
- Северская

Это сделано до сохранения в PostgreSQL, поэтому такие объекты не попадают в выдачу клиента и не создают записи в таблицах `properties` и `client_found_properties`.

## Районы и характеристики

Парсер пытается извлечь район из текста карточки, JSON-LD, адреса и описания.

Поддерживаются районы и распространенные обозначения:

- Прикубанский
- Карасунский
- Западный
- Центральный
- Фестивальный / ФМР
- Юбилейный / ЮМР
- Гидростроителей / ГМР

Для квартир дополнительно вытягиваются:

- площадь
- комнатность
- этаж
- этажность дома
- срок сдачи
- отделка

Для домов дополнительно вытягиваются:

- площадь дома
- площадь участка
- количество этажей
- спальни
- материал
- коммуникации

## Какие данные обычно пытаемся получить

- название объекта
- ссылка на источник
- цена
- площадь
- комнатность
- ЖК / КП
- район / локация
- этаж и этажность для квартир
- отделка для квартир
- участок, этажность, спальни, материал и коммуникации для домов
- описание
- изображения

## Что происходит, если источник не отдал данные

- если один сайт не ответил, поиск не падает целиком
- этот источник просто пропускается
- остальные источники продолжают поиск
- в выдачу попадают только те объекты, у которых есть достаточный минимум данных, прежде всего цена и ссылка

## Как фильтруются плохие карточки

Карточка отбрасывается, если:

- нет цены
- нет пригодной ссылки на объект
- объект относится не к городу Краснодар
- объект явно относится к другому городу Краснодарского края
- текст слишком короткий и не похож на карточку
- объект не проходит обязательные фильтры по бюджету

## Как фильтруются плохие названия

В нормализации title запрещены служебные слова, например:

- `Все новостройки`
- `Избранное`
- `Кабинет`
- `Отзывы`
- `Старт продаж`
- `Скоро в продаже`

Если источник отдает такой мусорный title, приложение собирает название из характеристик:

- для квартиры: `1-к квартира, 40.6 м² в ЖК «Nova Vita»`
- для дома: `Дом 128 м², участок 5 сот. в КП «Мелодия»`

## Как фильтруются логотипы и иконки

Изображение не используется, если URL содержит:

- `logo`
- `favicon`
- `icon`
- `sprite`
- `placeholder`
- `avatar`
- `social`
- `telegram`
- `vk`
- `whatsapp`

Также отбрасываются `.svg`.

## Ограничения

- поиск работает только по явно подключенным сайтам
- новые сайты автоматически не добавляются
- если сайт перестроил HTML, качество поиска по нему может временно ухудшиться
- демо-объявления не подмешиваются
