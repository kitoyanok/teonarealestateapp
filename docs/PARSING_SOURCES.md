# Источники парсинга

## Как устроен поиск

Поиск выполняет Python `search-service`.

Источники поиска не ищутся автоматически. Они заранее описаны в коде как адаптеры в:

- [apps/search-service/app/adapters/sources.py](/Users/like-shockpritotskaya-event/Documents/Зубач/Codex/realestate/apps/search-service/app/adapters/sources.py)

Каждый адаптер отвечает за конкретный сайт и приводит данные к единому формату `PropertyItem`.

Основная базовая логика парсинга находится в:

- [apps/search-service/app/adapters/base.py](/Users/like-shockpritotskaya-event/Documents/Зубач/Codex/realestate/apps/search-service/app/adapters/base.py)

## Что происходит при поиске

1. Node API формирует payload из параметров клиента.
2. Python `search-service` получает `POST /search`.
3. В зависимости от `propertyType` выбираются адаптеры квартир или домов.
4. Каждый адаптер идет на заранее заданные стартовые URL.
5. Из HTML и JSON-LD достаются карточки объектов.
6. Карточки нормализуются:
   - формируется title
   - очищается или генерируется description
   - фильтруются плохие картинки
   - цена и площадь приводятся к числам
7. Затем matcher проверяет соответствие параметрам клиента.
8. Только валидные объекты возвращаются в Node API.

## Какие данные обычно пытаемся получить

- название объекта
- ссылка на источник
- цена
- площадь
- комнатность
- ЖК / КП
- район / локация
- описание
- изображения

## Что происходит, если источник не отдал данные

- если один сайт не ответил, поиск не падает целиком
- этот источник просто пропускается
- остальные источники продолжают поиск
- в выдачу попадают только те объекты, у которых есть достаточный минимум данных, прежде всего цена и ссылка

## Как фильтруются плохие карточки

Карточка отбрасывается, если:

- нет цены
- нет пригодной ссылки на объект
- текст слишком короткий и не похож на карточку
- объект не проходит обязательные фильтры по бюджету

## Как фильтруются плохие названия

В нормализации title запрещены служебные слова, например:

- `Все новостройки`
- `Избранное`
- `Кабинет`
- `Отзывы`
- `Старт продаж`
- `Скоро в продаже`

Если источник отдает такой мусорный title, приложение собирает название из характеристик:

- для квартиры: `1-к квартира, 40.6 м² в ЖК «Nova Vita»`
- для дома: `Дом 128 м², участок 5 сот. в КП «Мелодия»`

## Как фильтруются логотипы и иконки

Изображение не используется, если URL содержит:

- `logo`
- `favicon`
- `icon`
- `sprite`
- `placeholder`
- `avatar`
- `social`
- `telegram`
- `vk`
- `whatsapp`

Также отбрасываются `.svg`.

## Ограничения

- поиск работает только по явно подключенным сайтам
- новые сайты автоматически не добавляются
- если сайт перестроил HTML, качество поиска по нему может временно ухудшиться
- демо-объявления не подмешиваются
