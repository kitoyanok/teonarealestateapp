# SEARCH QA CHECKLIST

## Цель

Проверить, что поиск учитывает параметры клиента, не подмешивает демо-объекты и корректно разделяет квартиры и дома.

## Автоматические проверки этой итерации

Добавлены unit-тесты:

- [apps/search-service/tests/test_matcher.py](/Users/like-shockpritotskaya-event/Documents/Зубач/Codex/realestate/apps/search-service/tests/test_matcher.py)
- [apps/search-service/tests/test_normalization.py](/Users/like-shockpritotskaya-event/Documents/Зубач/Codex/realestate/apps/search-service/tests/test_normalization.py)

Они проверяют:

- матчинг квартир по бюджету, комнатности, площади и району
- матчинг домов по бюджету, площади дома, участку и локации
- нормализацию мусорных title
- фильтрацию плохих изображений

## Как запускать тесты

```bash
cd apps/search-service
python3 -m unittest discover -s tests -v
```

## Ручная проверка квартир

| Сценарий | Параметры клиента | Ожидаемый результат | Фактический результат |
|---|---|---|---|
| Бюджет до 5 млн | квартира, до 5 млн | объекты дороже бюджета не попадают | проверить локально |
| Бюджет 5–8 млн | квартира, 5–8 млн | выдача остается в диапазоне | проверить локально |
| Студия | квартира, 0 комнат | попадают только студии | проверить локально |
| 1-комнатная | квартира, 1 комната | попадают только 1-к | проверить локально |
| 2-комнатная | квартира, 2 комнаты | попадают только 2-к | проверить локально |
| Площадь от 35 м² | квартира, min area 35 | мелкие объекты не попадают | проверить локально |
| Район | квартира, район Прикубанский | предпочтительно объекты из района | проверить локально |
| Срок сдачи | completion year filter | учитывается при наличии данных | проверить локально |

## Ручная проверка домов

| Сценарий | Параметры клиента | Ожидаемый результат | Фактический результат |
|---|---|---|---|
| Бюджет до 10 млн | дом, до 10 млн | дома дороже бюджета не попадают | проверить локально |
| Дом от 100 м² | houseAreaMin 100 | мелкие дома отсекаются | проверить локально |
| Участок от 4 сот | landAreaMin 4 | дома с меньшим участком не попадают | проверить локально |
| Локация | settlementNames Немецкая деревня | объекты из нужной локации | проверить локально |
| Коммуникации | communications | учитываются, если данные есть | проверить локально |
| Разделение типов | house search | квартиры не попадают | проверить локально |
| Разделение типов | apartment search | дома не попадают | проверить локально |

## Как проверять вручную

1. Запустить приложение:

```bash
npm run prod:up
```

2. Создать тестового клиента.
3. Заполнить конкретные параметры из таблиц выше.
4. Запустить поиск.
5. Проверить:
   - соответствие бюджета
   - соответствие типа объекта
   - адекватность названия
   - отсутствие мусорных фото в списке
   - наличие источника
6. При необходимости открыть запись в TablePlus и проверить сохранение:
   - `clients`
   - `properties`
   - `client_found_properties`
   - `shortlist_items`
   - `search_runs`
