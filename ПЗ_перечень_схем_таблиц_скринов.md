# Перечень схем, диаграмм, таблиц и скриншотов для ПЗ

Данный файл является рабочей картой по всем графическим и табличным материалам пояснительной записки. Его задача состоит в том, чтобы однозначно определить, какие рисунки, таблицы и скриншоты должны быть подготовлены для ПЗ по проекту `Тэона`, как они подписываются и что именно должно быть на них показано.

Файл синхронизирован с текущей структурой основного документа `Диплом_структура_ПЗ.md` и с отдельным файлом приложений `Приложения_ПЗ.md`.

# 0 Общие правила оформления по образцу Петруновского

## 0.1 Правило для рисунков

Для каждого рисунка в основном тексте ПЗ рекомендуется использовать одну и ту же схему подачи:

1. короткий вводный абзац перед рисунком;
2. подпись рисунка под изображением;
3. поясняющий абзац после рисунка со словами `В соответствии с рисунком ...`.

### Шаблон текста перед рисунком

`На рисунке N представлена ...`

### Шаблон подписи рисунка

`Рисунок N – ...`

### Шаблон текста после рисунка

`В соответствии с рисунком N, ...`

## 0.2 Правило для таблиц

Подпись таблицы ставится над таблицей. Формат подписи:

`Таблица N – ...`

Если таблица находится в приложении, используется буквенная нумерация:

- `Таблица А.1`;
- `Таблица В.1`;
- `Таблица Г.1`;
- `Таблица Д.1`;
- `Таблица Е.1`.

## 0.3 Правило для приложений

Приложения должны идти в следующем порядке:

- `Приложение А` — техническое задание;
- `Приложение Б` — руководство по стилю;
- `Приложение В` — программный код модулей;
- `Приложение Г` — тест-кейсы;
- `Приложение Д` — руководство пользователя;
- `Приложение Е` — руководство программиста.

## 0.4 Правило для формул

Формулы уместнее всего размещать в разделе алгоритмов, а не в приложениях. Для текущей ПЗ рекомендуется использовать формулы в подразделах:

- `2.6.2 Разработка алгоритма нормализации объявления`;
- `2.6.4 Разработка алгоритма расчета процента совпадения`.

Для проекта `Тэона` в основной текст логично включены следующие формулы:

- формула расчета стоимости одного квадратного метра;
- формула итогового процента совпадения;
- формула коэффициента совпадения по коммуникациям.

# 2 Кодовые рисунки для приложения В

## 2.1 Точный перечень рисунков и источников

### Рисунок В.1 – Авторизация риелтора и создание клиентской заявки

Рисунок В.1 рекомендуется делать составным из двух или трех вертикально расположенных фрагментов:

- файл `apps/api/src/data/realtors.ts`, фрагмент `seedRealtors`;
- файл `apps/api/src/routes/auth.ts`, фрагмент `authRouter.post("/login", ...)`;
- файл `apps/api/src/routes/clients.ts`, фрагменты `profileData`, `dbPayload`, `clientsRouter.post("/", ...)`.

### Рисунок В.2 – Запуск поиска и сохранение результатов поиска

Рисунок В.2 рекомендуется делать составным из двух фрагментов:

- файл `apps/api/src/routes/clients.ts`, фрагмент `clientsRouter.post("/:id/search", ...)`;
- файл `apps/api/src/services/searchService.ts`, фрагмент `runClientSearch`.

### Рисунок В.3 – Нормализация объявления и расчет процента совпадения

Рисунок В.3 рекомендуется делать составным из трех фрагментов:

### Рисунок В.4 – Формирование подборки, сообщения и сводных показателей

Рисунок В.4 рекомендуется делать составным из трех фрагментов:

- файл `apps/api/src/services/messageService.ts`, фрагмент `buildShareMessage`;
- файл `apps/api/src/routes/dashboard.ts`, фрагменты маршрутов `/summary` и `/activity`.

## 2.2 Как оформлять кодовые рисунки

В приложении В не следует показывать все файлы проекта подряд. По образцу Петруновского нужно выделить несколько крупных функциональных модулей и по каждому модулю показать один кодовый рисунок.

Перед рисунком рекомендуется дать краткий абзац о назначении модуля. Затем приводится таблица функций модуля. После таблицы вставляется кодовый рисунок с подписью.

После каждого рисунка нужен один короткий абзац:

`На рисунке В.N представлен фрагмент программной реализации модуля ..., который отвечает за ...`.

# 5 Формулы, которые должны сопровождать алгоритмы

## 5.1 Формула (1) – Расчет стоимости одного квадратного метра

Формулу следует размещать в подразделе `2.6.2` после описания извлечения цены и площади:

`C_{м²} = C / S`

Под формулой должно быть пояснение:

- `C_{м²}` — стоимость одного квадратного метра;
- `C` — полная стоимость объекта;
- `S` — общая площадь объекта.

## 5.2 Формула (2) – Итоговый процент совпадения объекта требованиям клиента

Формулу следует размещать в подразделе `2.6.4` после текста о весах критериев:

`M = (Σ(w_i * k_i) / Σw_i) * 100%`

Под формулой должно быть пояснение:

- `M` — итоговый процент совпадения;
- `w_i` — вес `i`-го критерия;
- `k_i` — коэффициент выполнения `i`-го критерия в диапазоне от `0` до `1`.

## 5.3 Формула (3) – Коэффициент совпадения по коммуникациям

Формулу следует размещать в подразделе `2.6.4` после формулы (2):

`k_{комм} = n_{совп} / n_{треб}`

Под формулой должно быть пояснение:

- `k_{комм}` — коэффициент совпадения по коммуникациям;
- `n_{совп}` — число совпавших коммуникаций;
- `n_{треб}` — число требуемых коммуникаций.
